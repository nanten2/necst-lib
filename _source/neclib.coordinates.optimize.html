neclib.coordinates.optimize¶

class DriveLimitChecker(limit, preferred_limit=None, *, unit=None, max_observation_size=<Quantity 5. deg>)[source]¶

Bases: object

Find optimum and safe angle to drive to.

The optimization includes:

  1. Apply drive range constraint

  2. Avoid direction unwrapping (360deg drive) during observation

  3. Over-180deg drive

Parameters

  • limit (Tuple[Union[int, float, ndarray, Quantity], Union[int, float, ndarray, Quantity]]) – Drive range limit, the optimization result never go out of this range.

  • preferred_limit (Tuple[Union[int, float, ndarray, Quantity], Union[int, float, ndarray, Quantity]]) – Drive range limit, which can be violated to continue observation or no other choice found.

  • unit (Union[str, Unit]) – Angular unit in which the parameters are given.

  • max_observation_size (Union[int, float, ndarray, Quantity]) – If separation between current and target coordinates is smaller than this value, direction unwrapping won’t occur, possibly violating preferred_limit, to not interrupt observation.

Return type

None

Examples

>>> checker = neclib.coordinates.DriveLimitChecker(
...     [-260 << u.deg, 260 << u.deg], [-250 << u.deg, 250 << u.deg]
... )
>>> checker.optimize(current=-200 << u.deg, target=170 << u.deg)
<Quantity -190. deg>
